    College & University sponsored.
    Final Year Project
    Involves: HTML5, CSS3, PHP, JavaScript, JQuery, WAMP, MySQL, IONIC 3, AngularJS, Google Firebase.
    Team Size - 5
    Duration - 180 days
    My Role - Project Lead
    I have designed the Logo too.

    “D2D (Day to Day) Smart Billing System” is developed that abets a person in everyday shopping in malls involving some automation in customarily practiced process. Malls and supermarkets are self-service shops offering a wide variety of food and household products. This system helps the customer in everyday shopping by minimizing the time required and increase shop productivity. The main objective is to provide a robust techno-savvy trolley with low-cost, high efficiency and easily adaptable system. Each techno-trolley is set up with a Raspberry Pi that is a unique identification device for that trolley. The camera scanner on the trolley will scan the purchased product and display it on LCD that is interfaced to the raspberry pi. The web-based application for trolleys and mobile-based application for baskets are designed to minimize the waiting time in “Queue” at the billing counters. At the billing counter, the confirmation of the total bill is done and the same is forwarded to the admin

    C.A.M.S.more_vert

    close

    Pre-final Year Project
    Centralized Attendance Management System for PWD on-site employees.
    Involves: Java 8, XML Android Studio 3.1.2, Google Firebase.
    Team Size - 5
    Duration - 60 days
    My Role - Project Lead
    I have designed the Logo too.

    Due to the current problems faced in the current attendance marking system that are high initial cost, portability, maintenance cost etc. We had developed a system that fulfills and overcomes all the drawbacks faced in the current trending attendance marking system.

    • ◉ The system provides attendance marking for on-field/site employees and accordingly assess their performance managed by the admin.

    • ◉ Our system is based on simple fingerprint or username authentication which will in turn take the user to the employee or admin login.

    • ◉ After successful authentication the employee or admin will be taken to the attendance marking page or scheduling working hours page respectively.

    • ◉ The employee can mark his/her attendance through an in time button and can also view his/her performance.

    • ◉ The attendance is only marked when the employee scans a QR code at the respective field for assigned duty.

    • ◉ The QR code scanning is to be done through the application itself which then will verify the location of the employee through location access and verify it with the location saved in the QR code.

    • ◉ Attendance will only be marked if the two locations are same.

    • ◉ The admin will monitor and assess the performance of each individual employee.

            

    String Theorymore_vert

    close

    Second Year Project
    Involves: C, Turbo C++
    Team Size - 5
    Duration - 60 days
    My Role - C Developer

    The problem statement was chosen from ACM ICPC 2016 and was named as String Theory.
    User is supposed to insert ( '' or "" ) quotes anywhere in a given string. Based on which we find the number of nesting levels inside a string. Each string was supposed to contain at-least one pair of '' or "" quotes.
    By canceling the quotes to it's same number we moved further to the center of string and thus get the number of nesting levels.
